# Design — implement-mcp-server-tools

## Architecture

The MCP server is a pure context provider. It has no analysis logic.

```
AI Agent
  │
  │  review_changes(ReviewOptions)
  ▼
MCP Server
  ├── analyzers/git.ts       → branch info + changed file paths
  ├── analyzers/project.ts   → framework, language, pkg manager, rules
  ├── analyzers/diff.ts      → parse diff, read full file contents
  └── validators/            → run lint / typecheck / test / build
  │
  │  ReviewContext
  ▼
AI Agent  →  reasons  →  writes inline comments to source files
```

## Types

### Input: ReviewOptions (existing)

```typescript
interface ReviewOptions {
  repoPath: string;
  baseBranch?: string;   // if omitted, MCP detects it
  runChecks?: boolean;   // default true
}
```

`generateArtifacts` and `outputDir` are removed — no artifacts are generated by the server.

### Output: ReviewContext (new type)

```typescript
interface ReviewContext {
  currentBranch: string;
  baseBranch: string;
  baseBranchReason: string;

  projectContext: ProjectContext;  // framework, language, packageManager, hasMonorepo
  projectRules: string;            // raw text collected from rule files in the repo

  changedFiles: ChangedFile[];
  checks: Checks;
}

interface ChangedFile {
  path: string;
  content: string;   // full file content
  diff: string;      // unified diff for this file only
}
```

`ReviewResult`, `ReviewSummary`, `ArtifactPaths` are removed from types — they described server-side output that no longer exists.

## Tool contract

### `review_changes`

```
Input:  ReviewOptions
Output: ReviewContext
```

Execution order:
1. `git.ts` — detect current branch, detect base branch, list changed files
2. `project.ts` — detect framework/language, collect project rules
3. `diff.ts` — parse diff per file, read full file content for each changed file
4. `validators/*` — run checks if `runChecks !== false`, capture pass/fail/not_configured
5. Assemble and return `ReviewContext`

If any validator is missing or fails to run, its check status is `not_configured` or `error`. Execution continues.

## Module responsibilities

### `analyzers/git.ts`

- Detect current branch name via `simple-git`
- Detect base branch: check for `main`, `master`, `develop` in order; use the one the current branch diverged from most recently
- Record `baseBranchReason` (e.g. `"'main' exists and current branch diverged 4 commits ago"`)
- Return list of changed file paths between current and base branch

### `analyzers/project.ts`

- Detect framework: check `package.json` deps for `next`, `react`, `vue`, `svelte`
- Detect language: presence of `tsconfig.json` → typescript; otherwise javascript; both → mixed
- Detect package manager: lockfile presence (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `bun.lockb`)
- Detect monorepo: presence of `packages/` or `apps/` directories with their own `package.json`
- Collect `projectRules`: read and concatenate content from these files if present:
  - `CLAUDE.md`
  - `CONVENTIONS.md`
  - `CONTRIBUTING.md` (only sections containing "convention", "naming", "style", "rule")
  - `eslint.config.js` / `.eslintrc` (file content, not evaluated)

### `analyzers/diff.ts`

- Use `simple-git` to get the unified diff between current and base branch
- Filter to frontend-relevant files: `.ts`, `.tsx`, `.js`, `.jsx`, `.css`, `.scss`, `.vue`, `.svelte`
- For each changed file, read full content from disk
- Return `ChangedFile[]`

### `validators/lint.ts`

- Detect ESLint (`eslint.config.js`, `.eslintrc*`) or Biome (`biome.json`)
- Run the detected linter in the `repoPath`
- Return `"passed"`, `"failed"`, `"not_configured"`, or `"error"`

### `validators/typecheck.ts`

- Detect `tsconfig.json`
- Run `tsc --noEmit`
- Return status + captured stderr on failure

### `validators/test.ts`

- Detect test runner from `package.json` scripts (`test`, `vitest`, `jest`)
- Run tests
- Return status

### `validators/build.ts`

- Detect build script from `package.json`
- Run build
- Return status

### `server.ts`

- Create MCP server with name `frontend-review` and version from `package.json`
- Register `review_changes` tool with Zod input schema matching `ReviewOptions`
- Start stdio transport

### `index.ts`

- Import and start the server

## Files to remove

The following stubs are out of scope and should be deleted:

- `src/tools/validate.ts`
- `src/analyzers/code.ts`
- `src/reporters/markdown.ts`
- `src/reporters/json.ts`
- `src/reporters/annotated.ts`
- `src/reporters/index.ts`

## Error handling

- Missing `repoPath` or non-existent path → throw with clear message
- Not a git repo → throw with clear message
- No changed files vs base branch → return `ReviewContext` with `changedFiles: []`
- Validator not found → `not_configured`, continue
- Validator crashes → `error`, continue
